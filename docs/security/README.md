# Docker Security Scan Reports

This directory contains Docker security scan reports generated by Trivy.

## Directory Structure

```
docs/security/
├── README.md                          # This file
├── baseline-scan-YYYY-MM-DD.txt      # Table format scan results
├── trivy-report-YYYY-MM-DD.json      # JSON format detailed report
└── trivy-results-YYYY-MM-DD.sarif    # SARIF format for GitHub Security
```

## File Types

### Baseline Scan (`.txt`)
Human-readable table format showing:
- Vulnerability ID (CVE)
- Severity level
- Affected package and version
- Fixed version (if available)
- Vulnerability description

**Use case**: Quick review and documentation

### JSON Report (`.json`)
Complete machine-readable vulnerability data including:
- Full CVE metadata
- CVSS scores and vectors
- Detailed descriptions
- References to security databases
- Package dependency trees

**Use case**: Automated processing, detailed analysis, archival

### SARIF Report (`.sarif`)
Static Analysis Results Interchange Format for GitHub Security:
- Integrated into GitHub Security tab
- Code scanning alerts
- Workflow integration

**Use case**: GitHub Security tab integration

## Generating Reports

### Manual Scan

```bash
# Run comprehensive scan
./scripts/run_docker_security_scan.sh

# Scan specific tag
./scripts/run_docker_security_scan.sh v1.2.3
```

### CI Automated Scan

Reports are automatically generated on every CI run and uploaded as artifacts:
1. Go to Actions → Select workflow run
2. Download "docker-security-report" artifact
3. Extract and review reports

## Report Retention

- **Local reports**: Keep last 3 months of monthly scans
- **CI artifacts**: 30 days retention (configured in workflow)
- **GitHub Security**: Permanent retention in Security tab
